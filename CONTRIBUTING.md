# Контрибьюция в t-invest-mcp

Спасибо за интерес! Проект — MCP-сервер для T-Invest API, ориентированный на
частных инвесторов. PR и issue приветствуются.

## Быстрый старт разработчика

```bash
git clone https://github.com/human-turn/t-invest-mcp && cd t-invest-mcp
npm install
npm run build     # TypeScript strict — сборка обязана быть чистой
```

Смоук-тест без токена и проверка с песочницей — см. [AGENTS.md](AGENTS.md)
(он же — инструкции для AI-агентов: структура, конвенции, инварианты).
Для живых проверок выпустите **sandbox-токен** в
[настройках Т-Инвестиций](https://www.tbank.ru/invest/settings/api/) и запускайте
с `TINKOFF_SANDBOX=true` — реальный счёт в этом режиме недостижим.

## Правила изменений

- **Не ослабляйте слои защиты** (read-only по умолчанию, elicitation-подтверждение
  сделок, path-guard файловых выгрузок) — PR с их ослаблением не принимаются.
- Стиль: TypeScript strict, без `any`; конвенции (деньги, enum'ы, время, `priceUnit`,
  `outputPath`) описаны в AGENTS.md.
- Перед PR: `npm run build` чистый + смоук-тест проходит; для затронутых tools —
  проверка в песочнице.
- Коммиты: краткое самодостаточное сообщение (русский или английский), одна
  логическая правка — один коммит.
- Процесс: форк → ветка от `main` → PR в `main`. Долгоживущих веток нет,
  `main` всегда релизопригоден.

## Баги и предложения

- Удобнее всего — команда `/t-invest:feedback <тема>` в Claude Code: соберёт
  репорт с диагностикой (без токена и личных данных) и предложит создать issue.
- Либо issue по шаблону. **Никогда не вставляйте токен, accountId и данные
  портфеля** — маскируйте.

## Релизы (для мейнтейнеров)
